<h2 style='margin-left:24.0pt'><a name="_Toc103598839">Differences between
Model P V6 and Model P2 V7 as well as Model S V6 and Model S2 V7</a></h2>

<h2 style='margin-left:0in'><o:p>&nbsp;</o:p></h2>

<p class=MsoBodyText style='margin-left:24.0pt'><span class=BodyTextChar>The
main difference between the Model P </span><span class=BodyTextChar><span
style='mso-fareast-font-family:"Calibri Light"'>V6 </span>and Model S </span><span
class=BodyTextChar><span style='mso-fareast-font-family:"Calibri Light"'>V6 </span>vs
the Model P</span><span class=BodyTextChar><span style='mso-fareast-font-family:
"Calibri Light"'>2 V7 </span>and Model S2</span><span class=BodyTextChar><span
style='mso-fareast-font-family:"Calibri Light"'> V7</span> is that there is an
additional Wi-Fi module incorporated internally for connection to the internet
via a Wi-Fi or via tethering to a mobile phone.<span style='mso-spacerun:yes'> 
</span>Additionally, the Model P2 and Model S2 have OTG capability on the USB C
power port allowing you to connect it directly to a host computer for data
transfer or other client hardware.<span style='mso-spacerun:yes'>  </span>Also,
we have upgraded the firmware from V1 – V2 allowing a more visually appealing
user experience.<span style='mso-spacerun:yes'>  </span>We have also added
language detection of the client and support multilingual content based on ISO
code identified folders in the data section.<span style='mso-spacerun:yes'> 
</span>We have added phone home capabilities to allow a central control server
to manage multiple Connectboxe’s in the field receiving statistical data on usage,
allowing control over all configuration features, and uploading of new content
to each unique Connectbox based on location.<o:p></o:p></span></p>

<p class=MsoBodyText style='margin-left:24.0pt'><span class=BodyTextChar><o:p>&nbsp;</o:p></span></p>

<p class=MsoBodyText style='margin-left:24.0pt'><span class=BodyTextChar>Another
difference between a V6 hardware and V7 hardware is that V6 has a slide power
switch on the side where V7 has a push for 3 second turn on and push for 6 second
turn off mode on the 1<sup>st</sup> switch closest to the LCD display.<span
style='mso-spacerun:yes'>  </span>The V6 <span class=GramE>switch</span> when
turned off will keep the power up on the unit for about 20 seconds before going
down to allow the system to save its state.<span style='mso-spacerun:yes'> 
</span>However, when you switch a unit off it will immediately stop
transmission over the <span class=SpellE>Wifi</span>.<o:p></o:p></span></p>

for creating a password is the same as creating a good Passphrase.</p>

it will look at.</p>

<h2 style='text-indent:21.0pt'><a name="_Toc103598846">Enabling SSH for the

<p class=MsoBodyText><o:p>&nbsp;</o:p></p>

<p class=MsoBodyText style='margin-left:42.0pt'>There is a way to enable SSH
into the unit on boot.<span style='mso-spacerun:yes'>  </span>This is
accomplished by plugging in a USB Flash drive with the following structure:</p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-tab-count:3'>                                    </span>root
directory:<span style='mso-tab-count:1'> </span>&#9472;.connectbox<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-spacerun:yes'>    </span><span
style='mso-spacerun:yes'>                                                                </span>&#9492;&#9472;&#9472;<span

<p class=MsoBodyText style='margin-left:42.0pt'><b><o:p>&nbsp;</o:p></b></p>

<p class=MsoBodyText style='margin-left:42.0pt'>Note that the directory is
(dot) connectbox with a single file enable-<span class=SpellE>ssh</span> that
can be blank or empty.</p>

<p class=MsoBodyText style='margin-left:42.0pt'><o:p>&nbsp;</o:p></p>

<p class=MsoBodyText style='margin-left:42.0pt'>As soon as this is plugged in

<p class=MsoBodyText style='margin-left:42.0pt'><o:p>&nbsp;</o:p></p>

<p class=MsoBodyText style='margin-left:24.0pt'><a name="_Toc103598847"><span

<p class=MsoBodyText style='margin-left:24.0pt'><o:p>&nbsp;</o:p></p>

<p class=MsoBodyText style='margin-left:42.0pt'>It is possible that interim patches

</span>Using a clean USB flash drive load the patch code into the following
directory structure:</p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-tab-count:3'>                                    </span>root
directory:<span style='mso-tab-count:1'> </span>&#9472;.connectbox - upgrade<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-spacerun:yes'>                                                                 
</span><span style='mso-spacerun:yes'>                  </span><span
style='mso-spacerun:yes'>  </span>&#9500;&#9472;&#9472; Prerun<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-spacerun:yes'>                                                               
</span><span style='mso-spacerun:yes'>                  </span><span
style='mso-spacerun:yes'>    </span>&#9500;&#9472;&#9472; Postrun<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span

style='mso-spacerun:yes'>                                         </span><span

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-spacerun:yes'>                                                                                     
</span>&#9500;&#9472;&#9472; CM4version<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt;mso-no-proof:yes'><span
style='mso-spacerun:yes'>                </span><span
style='mso-spacerun:yes'>                                              </span><span

<p class=MsoBodyText><o:p>&nbsp;</o:p></p>

<p class=MsoBodyText style='margin-left:40.5pt'>When this USB flash drive is inserted,
the operating system will detect the upgrade folder and look for the upgrade.py
file.<span style='mso-spacerun:yes'>  </span>If it exists, it will then copy
the upgrade code/directories to the internal memory and execute the
upgrade.<span style='mso-spacerun:yes'>  </span>If successful it will then will
create an Upgrade xxxzxxxx.log file in the uSD directory /<span class=SpellE>usr</span>/local/connectbox<span

directory.<span style='mso-spacerun:yes'>   </span>If the upgrade <span
class=SpellE>failes</span>, you will get a message on the display indicating
failure and an error code.<span style='mso-spacerun:yes'>  </span>Contact <a
href="mailto:help@connectbox.org">help@connectbox.org</a> for additional
assistance.<span style='mso-spacerun:yes'>  </span>If the upgrade was
successful, you will receive a notice of success on the screen and you can
remove and reformat the USB flash drive you used.</p>

<p class=MsoNormal style='margin-left:22.5pt'><a name="_Toc103598850"><span
class=Heading2Char><span style='font-size:13.0pt'>Linking your Connectbox to
Control Host</span></span></a>:</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>The connectbox can be run independently in the field or it
can be connected periodically to a control host system for an organization or
for reporting of statistical information for media being used from another
vendor.<span style='mso-spacerun:yes'>  </span>The host management system allows
for content packages to be prepared for download to specific Connectboxe’s.<span
style='mso-spacerun:yes'>  </span>It also allows for return uploading of
statistical information from the connectbox in the field.<span
style='mso-spacerun:yes'>  </span>This is done by connecting the connectbox to
the internet periodically either through an access point with internet access,
or through a mobile phone with a data plan tethered to the connectbox. (Note an
access point can simply be a local sharing of your mobile data plan as a
hotspot for the connectbox to connect to.)</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>The following diagram outlines the general approach for
remote Connectboxe’s being integrated into the Control Host:</p>

<p class=MsoNormal>Here media is prepared by the content package creator
through the package manager interface of the Control Host system.<span
style='mso-spacerun:yes'>  </span>These packages are then queued for download
to the individual Connectboxe’s based on unique ID of the box and
location/local administrator.<span style='mso-spacerun:yes'>  </span>When the
local connectbox has an internet connection either through an Access Point (AP)
or by the local administrator connecting it to one, the media package will be
downloaded and installed.<span style='mso-spacerun:yes'>  </span>Simultaneously
it will have its media usage statistics sent to the Control Host system.<span
style='mso-spacerun:yes'>  </span>The Content package creator/or Statistical
analyst can then aggregate the information of media usage across units to
provide reporting to media vendors that created the content.<span
style='mso-spacerun:yes'>  </span>There are specific role responsibilities in
this model.</p>

<p class=MsoListParagraph style='margin-left:.5in;mso-list:l8 level1 lfo9'><![if !supportLists]><span

</span></span><![endif]>Local Administrator, their job is to secure the unit,
manage the units’ capabilities locally, connect it periodically to the internet
for retrieval of new content as well as reporting of content usage.</p>

<p class=MsoListParagraph style='margin-left:.5in;mso-list:l8 level1 lfo9'><![if !supportLists]><span
style='mso-list:Ignore'>2)<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>The Media content creator job is to prepare the
packages of media that each unique unit needs as well as to oversee the
ensuring of periodic reporting in of the individual boxes from the field
through the return of data from the units.</p>

<p class=MsoListParagraph style='margin-left:.5in;mso-list:l8 level1 lfo9'><![if !supportLists]><span
style='mso-list:Ignore'>3)<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>The Statistician who’s job it is to aggregate the
statistical information from the Host Controller system and report back to the
media providers and internally to the organization.</p>

<p class=MsoListParagraph style='margin-left:.5in;mso-list:l8 level1 lfo9'><![if !supportLists]><span
style='mso-list:Ignore'>4)<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span><![endif]>The System Administrators job is to setup and mange the
Control Host System.<span style='mso-spacerun:yes'>  </span>This is done either
virtually on an Amazon Web Server or other hosting provider, or locally on
physical hardware.<span style='mso-spacerun:yes'>  </span>The are to ensure the
ongoing maintenance and patching of software to ensure security of the system.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>To facilitate the deployment of the Connectboxe’s for an organization,
we have prepared the Host Controller software system installation packages for
AWS servers as well as other providers like Digital Ocean through Ansible
script files.<span style='mso-spacerun:yes'>  </span>This will create the
structure on the server and provision it.<span style='mso-spacerun:yes'> 
</span>Additional DNS records management may be needed to ensure that the
ConnectBox’s and system users have easy access to this control system.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal><b>Please refer to the documentation on setup and

needed in setting up the individual Connectboxe’s.</p>
